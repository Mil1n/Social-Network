## 💬 Чаты и сообщения

- **Статусы сообщений**: отправлено, доставлено, прочитано.
- **Typing indicator**: «Алекс печатает…».
- **Ответы на сообщения**: reply preview внутри bubble.
- **Закреплённые сообщения**: важная информация сверху чата.
- **Поиск по истории чата**: поиск текста по текущему диалогу.
- **Реакции**: 👍 ❤️ 😂 🔥 👀 для сообщений и постов.
- **Пересылка сообщений**: отправка сообщения в другой чат или канал.
- **Черновики**: сохранять недописанный текст в каждом чате.
- **Голосовые сообщения**: запись аудио в браузере через MediaRecorder API.

## 📣 Каналы и социальная лента

- **Комментарии к постам**: обсуждения под публикациями.
- **Отложенные публикации**: планирование постов на дату и время.
- **Черновики постов**: сохранение незавершённых публикаций.
- **Публичные и приватные каналы**: разные уровни доступа.
- **Пригласительные ссылки**: `join/:inviteCode` для групп и каналов.
- **Закреплённые посты**: важные публикации вверху канала.
- **Статистика канала**: просмотры, подписчики, реакции, популярные посты.

## 👤 Профили и социальный граф

- **Фотография профиля** и обложка.
- **Статус пользователя**: online, offline, last seen.
- **Контакты/друзья**: запросы, подтверждения, список контактов.
- **Блокировка пользователей**: запрет сообщений и просмотра профиля.
- **Настройки приватности**: кто видит bio, online, каналы и контакты.
- **Публичная страница профиля**: короткая ссылка на пользователя.

## 🛡️ Модерация и безопасность

| Идея | Что даёт |
| --- | --- |
| Очередь жалоб с изменением статуса | Модераторы смогут закрывать, отклонять и эскалировать жалобы |
| Бан/разбан пользователей | Базовое управление нарушителями |
| Мут в чатах | Временное ограничение активности |
| Audit log фильтры | Быстрый поиск действий администраторов |
| Rate limiting | Защита от спама и brute force |
| bcrypt/Argon2 | Безопасное хранение паролей |
| Session expiration | Сессии не будут жить бесконечно |
| CSRF/XSS hardening | Безопаснее для публичного запуска |

## ⚡ Realtime

Текущий MVP работает через обычные REST-запросы. Следующий большой шаг — realtime.

Рекомендуемый путь:

1. Добавить **Server-Sent Events** для уведомлений и новых сообщений.
2. Затем перейти на **WebSocket**, когда появятся typing indicators, presence и звонки.
3. Для масштабирования вынести события в **Redis Pub/Sub** или очередь сообщений.

Что появится после realtime:

- новые сообщения без обновления страницы;
- live-уведомления;
- online/offline статусы;
- typing indicator;
- быстрые реакции;
- live-модерация.

## 🗄️ Переход на production-хранилище

Сейчас данные хранятся в JSON-файле. Для реального продукта лучше перейти на:

- **PostgreSQL** — пользователи, чаты, сообщения, каналы, роли, жалобы;
- **Redis** — кеш, rate limiting, online presence, pub/sub;
- **S3-compatible storage** — изображения, видео, документы, голосовые сообщения;
- **Meilisearch/OpenSearch** — быстрый поиск по сообщениям, постам и пользователям.

## 💰 Монетизация

Этичные варианты, которые не ломают доверие:

- premium-профили;
- расширенное хранилище файлов;
- платные приватные каналы;
- донаты авторам;
- корпоративные workspace-тарифы;
- white-label версия для закрытых сообществ.

Лучше избегать агрессивной рекламы и продажи пользовательских данных — для личной социальной сети доверие важнее краткосрочной выручки.

## 🧭 Рекомендуемый следующий update

Если выбирать только один следующий шаг, лучше сделать **полноценную систему сообщений уровня Telegram Lite**:

1. Добавить редактирование и удаление сообщений.
2. Добавить реакции.
3. Добавить reply на сообщение.
4. Добавить индикатор непрочитанных сообщений.
5. Добавить нормальный logout.
6. Добавить toast-уведомления вместо `alert()`.
7. Добавить визуально активный выбранный чат.

Это даст максимум ощутимой пользы для пользователя без сложной backend-перестройки.
