# Промт для проектирования личной социальной сети по типу Telegram

Скопируй этот промт в ИИ-ассистента или используй как техническое задание для команды разработки.

```text
Ты — senior product manager, system architect, UX/UI designer и lead full-stack engineer. Помоги спроектировать и подготовить к разработке полноценную личную социальную сеть по типу Telegram, но с акцентом на приватность, контроль владельца, масштабируемость и современный пользовательский опыт.

Контекст проекта:
Я хочу создать собственную личную социальную сеть/мессенджер, похожую по уровню удобства на Telegram. Это не просто чат, а закрытая или полуоткрытая социальная платформа, где пользователи могут общаться, публиковать контент, создавать каналы/группы, обмениваться файлами, вести профили и пользоваться безопасной инфраструктурой.

Главная цель:
Разработать подробную концепцию, архитектуру, roadmap и техническое задание для MVP и последующего развития продукта.

Обязательные разделы ответа:

1. Видение продукта
- Опиши идею продукта простыми словами.
- Сформулируй ценностное предложение: почему пользователи захотят пользоваться моей сетью, если уже есть Telegram, WhatsApp, Discord, VK и другие платформы.
- Предложи 3–5 возможных позиционирований: личная закрытая сеть, семейная сеть, комьюнити-платформа, корпоративная сеть, сеть для авторов и подписчиков.

2. Целевая аудитория
- Опиши основные сегменты пользователей.
- Для каждого сегмента укажи боли, сценарии использования и ключевые функции.
- Раздели пользователей на роли: владелец платформы, администратор, модератор, обычный пользователь, автор канала, участник группы.

3. Функциональность MVP
Подробно распиши минимальную первую версию продукта. В MVP обязательно включи:
- регистрацию и вход;
- профиль пользователя;
- контакты и поиск людей;
- личные сообщения;
- групповые чаты;
- каналы/ленты публикаций;
- отправку изображений, видео, документов и голосовых сообщений;
- уведомления;
- базовую модерацию;
- настройки приватности;
- админ-панель;
- систему жалоб и блокировок.

Для каждой функции опиши:
- пользовательский сценарий;
- основные экраны;
- backend-логику;
- данные, которые нужно хранить;
- ограничения MVP;
- что можно отложить на будущие версии.

4. Расширенные функции после MVP
Предложи развитие продукта по этапам:
- end-to-end шифрование;
- звонки и видеозвонки;
- stories/короткие публикации;
- реакции и комментарии;
- боты и мини-приложения;
- платные подписки;
- донаты авторам;
- рекомендации контента;
- поиск по сообщениям и файлам;
- облачное хранилище;
- веб-версия, iOS и Android;
- публичные ссылки-приглашения;
- роли и права в группах;
- интеграции с внешними сервисами.

5. UX/UI
- Предложи структуру интерфейса, похожую по удобству на Telegram, но с собственной идентичностью.
- Опиши основные экраны: вход, список чатов, чат, профиль, канал, группа, настройки, админ-панель.
- Дай рекомендации по навигации, цветовой палитре, типографике, состояниям загрузки, пустым состояниям и ошибкам.
- Опиши, как сделать интерфейс простым для обычных пользователей и мощным для администраторов.

6. Техническая архитектура
Спроектируй современную архитектуру приложения:
- frontend;
- backend;
- база данных;
- realtime-сообщения;
- хранение файлов;
- очередь задач;
- уведомления;
- поиск;
- кэширование;
- аналитика;
- логирование и мониторинг;
- админ-панель.

Предложи несколько стеков на выбор:
- быстрый MVP;
- масштабируемый production-вариант;
- максимально приватный self-hosted-вариант.

Для каждого стека укажи плюсы, минусы, примерные технологии и причины выбора.

7. Модель данных
Опиши основные сущности:
- User;
- Profile;
- Session;
- Chat;
- Message;
- Attachment;
- Group;
- Channel;
- Post;
- Comment;
- Reaction;
- Notification;
- Report;
- Block;
- Role;
- Permission;
- Invite;
- AuditLog.

Для каждой сущности перечисли ключевые поля и связи с другими сущностями.

8. Безопасность и приватность
Разработай отдельный раздел по безопасности:
- хэширование паролей;
- двухфакторная аутентификация;
- защита сессий;
- rate limiting;
- антиспам;
- модерация;
- шифрование данных;
- безопасная загрузка файлов;
- защита от XSS, CSRF, SQL injection, IDOR;
- аудит действий администраторов;
- резервные копии;
- удаление аккаунта и данных пользователя.

Объясни, какие меры обязательны для MVP, а какие можно внедрять позже.

9. Масштабирование
Опиши, как проектировать систему, чтобы она выдерживала рост:
- от 100 пользователей;
- до 10 000 пользователей;
- до 1 000 000 пользователей.

Для каждого этапа укажи, что нужно менять в инфраструктуре, базе данных, хранении файлов, realtime-сервисе и мониторинге.

10. Монетизация
Предложи этичные способы монетизации:
- premium-аккаунты;
- расширенное облачное хранилище;
- платные приватные каналы;
- донаты;
- подписки на сообщества;
- white-label версии;
- корпоративные тарифы.

Укажи, какие модели подходят для личной социальной сети, а какие лучше не использовать, чтобы не разрушить доверие пользователей.

11. Roadmap
Составь roadmap по этапам:
- исследование и прототипирование;
- дизайн;
- MVP;
- закрытая beta;
- публичный запуск;
- развитие продукта;
- масштабирование.

Для каждого этапа укажи примерную длительность, результат, риски и критерии готовности.

12. Команда и роли
Опиши, какая команда нужна для создания продукта:
- product manager;
- UX/UI designer;
- frontend developer;
- backend developer;
- mobile developer;
- DevOps engineer;
- QA engineer;
- security specialist;
- community/moderation manager.

Также предложи минимальную команду для старта, если бюджет ограничен.

13. Оценка сложности и бюджета
Дай примерную оценку сложности:
- no-code/low-code прототип;
- MVP силами небольшой команды;
- production-версия;
- мобильные приложения;
- поддержка и инфраструктура.

Оцени не точную цену, а диапазоны усилий, риски и факторы, влияющие на стоимость.

14. Юридические и организационные вопросы
Опиши, что нужно предусмотреть:
- пользовательское соглашение;
- политика конфиденциальности;
- правила сообщества;
- обработка персональных данных;
- возрастные ограничения;
- жалобы на контент;
- удаление незаконного контента;
- хранение логов;
- соответствие требованиям страны, где будет работать сервис.

15. Итоговый план действий
В конце составь конкретный пошаговый план на первые 30 дней:
- какие решения принять;
- какие документы подготовить;
- какие макеты сделать;
- какой прототип собрать;
- какие технологии выбрать;
- какие риски проверить;
- что должно быть готово к концу месяца.

Требования к стилю ответа:
- Пиши структурированно, глубоко и практично.
- Не ограничивайся общими словами.
- Давай конкретные рекомендации и варианты выбора.
- Если есть несколько подходов, сравни их в таблицах.
- Указывай риски и компромиссы.
- Думай как человек, который реально собирается запускать продукт, а не просто фантазирует.
- Не предлагай копировать Telegram один в один: нужна собственная социальная сеть с похожим уровнем удобства, но со своей концепцией.
```
