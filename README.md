# 🚀 Private Social Network MVP

> 💬 Личная социальная сеть/мессенджер в стиле Telegram для локального запуска, быстрых демонстраций и дальнейшего развития в полноценный продукт.

## ✨ О проекте

Private Social Network MVP — это минимальная, но рабочая основа для собственной социальной сети: пользователи регистрируются, создают профили, находят друг друга, общаются в чатах, подписываются на каналы, публикуют посты, получают уведомления и отправляют жалобы на контент.

💡 Проект специально сделан без внешних зависимостей, чтобы его было легко запустить, изучить, расширить или переписать под production-архитектуру.

## 🌟 Вайб проекта

- 🚀 Быстро запустить и сразу увидеть результат в браузере.
- 🔐 Держать данные под своим контролем, а не зависеть от чужой платформы.
- 💬 Построить основу для чатов, каналов и личного комьюнити.
- 🧪 Экспериментировать с продуктовой идеей без сложной инфраструктуры.
- 🏗️ Постепенно превратить MVP в production-ready социальную сеть.

## ⚡ Быстрый старт

### 1. 🧰 Требования

- Node.js `20+`
- npm

### 2. 📦 Установка

```bash
npm install
```

> В проекте нет внешних зависимостей, поэтому установка нужна в основном для стандартного npm workflow.

### 3. 🌐 Запуск в браузере на этом компьютере

```bash
npm start
```

Откройте приложение в браузере:

```text
http://localhost:3000
```

По умолчанию сервер слушает `0.0.0.0`, поэтому приложение доступно не только с этого компьютера, но и с других устройств в вашей локальной сети.

### 4. 📱 Доступ с телефона или другого компьютера в Wi‑Fi/LAN

1. Запустите сервер:

   ```bash
   npm run start:lan
   ```

2. Узнайте IP-адрес компьютера, где запущен проект.

   Linux/macOS:

   ```bash
   hostname -I
   ```

   Windows PowerShell:

   ```powershell
   ipconfig
   ```

3. Откройте в браузере телефона или другого устройства:

   ```text
   http://<IP-адрес-компьютера>:3000
   ```

   Например:

   ```text
   http://192.168.1.25:3000
   ```

🧯 Если браузер не открывает сайт, проверьте firewall, VPN, одинаковую Wi‑Fi/LAN сеть и что порт `3000` не заблокирован.

### 5. 🔒 Локальный режим только для текущего компьютера

Если нужно запретить доступ из локальной сети и открыть приложение только на текущем устройстве:

```bash
npm run start:local
```

🎉 Первый зарегистрированный пользователь автоматически получает роль администратора.

### 6. 🩺 Проверка работоспособности

После запуска можно проверить health endpoint:

```text
http://localhost:3000/api/health
```

Ожидаемый ответ:

```json
{ "status": "ok" }
```

### 7. ✅ Тесты

```bash
npm test
```

## 🕹️ Как пользоваться

1. 👑 Зарегистрируйте первого пользователя — он станет администратором.
2. 🧑‍🤝‍🧑 Создайте второго пользователя в другом браузере, профиле браузера или после очистки `localStorage`.
3. 🔎 Найдите пользователя через поиск и скопируйте его ID.
4. 💬 Создайте чат и добавьте ID участника через запятую.
5. ✉️ Отправьте сообщение или URL-вложение.
6. 📣 Создайте канал, подпишитесь другим пользователем и опубликуйте пост от имени владельца.
7. 🚨 Отправьте жалобу на объект по его ID.
8. 🛡️ Нажмите «Открыть админ-панель» от имени администратора, чтобы посмотреть пользователей, жалобы и audit log.

## 🔌 API

Все защищённые эндпоинты требуют заголовок:

```text
Authorization: Bearer <token>
```

| Метод | Путь | Назначение |
| --- | --- | --- |
| `POST` | `/api/register` | Регистрация пользователя |
| `POST` | `/api/login` | Вход и получение токена |
| `GET` | `/api/me` | Текущий пользователь |
| `PATCH` | `/api/profile` | Обновление профиля |
| `GET` | `/api/users?q=` | Поиск пользователей |
| `GET` | `/api/chats` | Список чатов пользователя |
| `POST` | `/api/chats` | Создание чата |
| `GET` | `/api/chats/:id/messages` | История сообщений |
| `POST` | `/api/chats/:id/messages` | Отправка сообщения |
| `GET` | `/api/channels` | Список каналов |
| `POST` | `/api/channels` | Создание канала |
| `POST` | `/api/channels/:id/subscribe` | Подписка на канал |
| `GET` | `/api/channels/:id/posts` | Посты канала |
| `POST` | `/api/channels/:id/posts` | Публикация поста владельцем канала |
| `POST` | `/api/reports` | Создание жалобы |
| `GET` | `/api/notifications` | Уведомления пользователя |
| `GET` | `/api/admin` | Админ-обзор пользователей, жалоб и audit log |

## 🛠️ Технологии

- **Node.js HTTP server** — без Express и других зависимостей.
- **ES Modules** — современный формат модулей JavaScript.
- **JSON-файл** — простое локальное хранилище для MVP.
- **Vanilla HTML/CSS/JS** — лёгкий frontend без сборщика.
- **node:test** — встроенный тестовый раннер Node.js.

## ⚙️ Переменные окружения

| Переменная | По умолчанию | Описание |
| --- | --- | --- |
| `PORT` | `3000` | Порт HTTP-сервера |
| `HOST` | `0.0.0.0` | Адрес прослушивания: `0.0.0.0` для локальной сети или `127.0.0.1` только для текущего компьютера |
| `DB_FILE` | `data/db.json` | Путь к JSON-файлу базы данных |

Пример:

```bash
HOST=0.0.0.0 PORT=8080 DB_FILE=data/local.json npm start
```

## 🚧 Ограничения текущего MVP

Эта версия предназначена для локальной демонстрации и развития идеи. Перед production-запуском обязательно доработайте:

- password hashing через Argon2 или bcrypt вместо демонстрационного SHA-256;
- HTTPS, secure cookies, refresh tokens и истечение сессий;
- PostgreSQL или другую полноценную базу данных;
- Redis для кеша, rate limiting и очередей;
- WebSocket или SSE для realtime-сообщений;
- объектное хранилище для файлов и изображений;
- проверку и антивирусную обработку загрузок;
- роли, права, бан пользователей и модераторские действия;
- антиспам, throttling и защиту от brute force;
- XSS/CSRF/IDOR hardening;
- резервные копии и миграции;
- юридические документы: privacy policy, terms of service, правила сообщества.

## 🛡️ Безопасность

⚠️ Не используйте текущую версию как публичный production-сервис без доработок. Код показывает продуктовую механику и базовую архитектуру, но не заменяет полноценный security review.

Если вы разворачиваете проект даже для небольшой закрытой группы, минимум включите HTTPS, сильное хэширование паролей, rate limiting, безопасные cookies/сессии и регулярные резервные копии.
